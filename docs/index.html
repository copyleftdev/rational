How this was built

Two engines. The model never does the arithmetic.

Cost models are exactly where floating-point lies and language models fumble — tiny per-token prices, enormous token counts, compounding rates. So the labor is split.

01Research agents gather

Live web search returns current prices and benchmarks with source URLs and observation dates. Half of 2026's models are already legacy — memory won't do.

02An exact kernel computes

Every figure runs through agent-calc in exact rationals — 3⁄5 × 1,000,000 = 600000, not 599999.99998. LP, NPV, algebra, statistics, all typed.

03You re-run it

Each chapter is a script; each input price lives one file away. make all → bit-identical numbers. That reproducibility is the whole argument.

Chapter 02 · self-host

Renting GPUs almost never wins

It's not a dollar break-even — it's a utilization one. A node costs the same idle or busy. Above 100%, self-hosting loses even at full tilt. Only owned Blackwell, kept busy, beats a $0.20/1M API.

Chapter 04 · cascade

Cheap first, escalate rarely

Run a cheap model on everything; escalate the residual failures to a frontier model. Three tiers reach 99.5% coverage at a sixth of frontier-only cost — because 95.6% of tasks never reach the top tier.

Chapter 05 · commit vs wait

Never lock in a falling price

Commodity token prices fall ~80%/yr — faster than any reservable discount. So for commodity models, waiting always wins; commit only where prices are sticky (frontier), and never for three years.

Chapters 06 – 07 · the agent loop

Why agents are quadratic — and cacheable

An agent re-sends its growing context every step, so cumulative input is O(K²). Caching halves the constant; only compaction breaks the curve. And caching pays from the 2nd–3rd call — which is why 63.75% of the workload bills at the cached rate.
